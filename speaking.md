# Speaking — GVoiceBot

## Введение

Мой проект называется GVoiceBot — это голосовой ассистент с графическим интерфейсом, написанный на Python. Он умеет распознавать речь, обрабатывать команды и отвечать голосом. Также поддерживается текстовый ввод — можно печатать команды прямо в окне.

---

## 1. Точка входа — main.py

Программа запускается с файла `main.py`. Здесь происходят две вещи: создаётся объект `Assistant` со списком всех скиллов, и вызывается функция `run()` из модуля `gui/app.py`.

Функция `process_query` — это единственный публичный интерфейс логики. И голосовой поток, и текстовый ввод вызывают именно её. Она возвращает кортеж: что произнести и что показать в чате.

```python
_assistant = Assistant([
    HistorySkill(),
    GreetingSkill(),
    TimeSkill(),
    ...
])

def process_query(text: str) -> tuple[str | None, str]:
    return _assistant.process(text)
```

---

## 2. Запуск GUI — gui/app.py

Функция `run()` сначала загружает `.env` файл с API-ключами через `load_env()`, затем создаёт `QApplication`, применяет таблицу стилей в теме Catppuccin Macchiato, устанавливает шрифт и показывает главное окно.

---

## 3. Главное окно — gui/main_window.py

`MainWindow` строит интерфейс из нескольких виджетов:

- `SidebarWidget` — боковая панель с кнопками настроек и выхода
- `StatusWidget` — статус-бар: WAKE, REC, THINKING, SPEAKING
- `ChatWidget` — область отображения диалога
- Поле ввода текста и кнопка Send
- Кнопка Activate — запускает прослушивание вручную, без wake-word

После построения UI вызывается `_start_thread()`, который создаёт и запускает `AssistantThread`.

---

## 4. Голосовой цикл — gui/assistant_thread.py

`AssistantThread` наследует `QThread` и работает в фоне, не блокируя интерфейс. В методе `run()` запускается бесконечный цикл.

На каждой итерации поток:
1. Переходит в состояние WAKE
2. Вызывает `listen_for_wake_word()` — ждёт фразу «Voice»
3. Если wake-word обнаружен — переходит в состояние REC и вызывает `listen()`
4. Передаёт текст в `process_query()`
5. Получает `(voice, display)` — показывает `display` в чате, произносит `voice`
6. Сохраняет оба сообщения в базу данных

Кнопка Activate устанавливает событие `threading.Event`, которое на следующей итерации пропускает шаг ожидания wake-word.

---

## 5. Распознавание речи — stt.py

Модуль содержит четыре функции.

`_record(duration)` — записывает аудио через `sounddevice`. Использует нативную частоту дискретизации устройства, чтобы избежать проблем с PipeWire.

`_is_silent(audio)` — вычисляет RMS сигнала. Если он ниже порога `SILENCE_THRESHOLD`, запись считается тишиной и не отправляется на транскрипцию.

`_transcribe(audio_path)` — отправляет WAV-файл в Groq API, модель Whisper large-v3, получает текст.

`listen_for_wake_word()` — записывает 3 секунды, проверяет наличие слова «voice». Если пользователь сказал «Voice, what time is it» — возвращает инлайн-команду, и второй шаг записи пропускается.

---

## 6. Обработка команды — assistant.py

`Assistant.process()` перебирает список скиллов и вызывает `can_handle(text)` у каждого. Первый скилл, который вернул `True`, обрабатывает запрос через `execute(text)`.

Если ни один скилл не совпал — возвращается стандартный ответ «I did not understand the request».

Ответ скилла может быть двух форматов:
- Обычная строка — произносится и показывается как есть
- Строка с префиксом `\x00voice\x00display` — тогда голосом произносится `voice`, а в чат идёт `display`. Это используется в `HistorySkill`, чтобы не зачитывать весь список сообщений.

---

## 7. Скиллы — skills/

Все скиллы наследуют абстрактный класс `Skill` из `skills/base.py`. Он определяет два члена:

- `keywords` — список ключевых слов
- `can_handle(text)` — проверяет, есть ли любое ключевое слово в тексте по границе слова через регулярное выражение `\b`
- `execute(text)` — абстрактный метод, который нужно реализовать

Чтобы добавить новый скилл, достаточно создать файл в `skills/`, унаследоваться от `Skill`, задать `keywords` и реализовать `execute()`, а затем добавить одну строку в `main.py`.

Реализованные скиллы:

**GreetingSkill** — отвечает на приветствия: hello, hi, hey и другие.

**TimeSkill / DateSkill** — возвращают текущее время и дату. `strftime` используется с форматом `%A, %B %-d, %Y`, что гарантирует английские названия независимо от системной локали.

**WeatherSkill** — извлекает название города регулярным выражением, запрашивает OpenWeatherMap API и возвращает температуру и описание погоды. Обрабатывает таймаут, ошибку сети и отсутствие API-ключа.

**CalculatorSkill** — заменяет словесные операторы (plus, minus, times) на символы, фильтрует строку белым списком символов и вызывает `eval()`. Обрабатывает деление на ноль и некорректный ввод.

**ReminderSkill** — парсит длительность регулярным выражением, создаёт `threading.Timer`, который через заданное время вызывает `tts.speak()`.

**HistorySkill** — показывает историю диалогов из базы данных. `show history` — последние 20 сообщений, `history today` — только сегодняшние, `clear history` — очищает базу. Отвечает голосом «Here is the history», а полный список выводит только в чат.

---

## 8. Синтез речи — tts.py

`speak(text)` работает в три шага:

1. `edge_tts.Communicate(text, VOICE).save(tmp_path)` — асинхронно генерирует MP3 через публичный эндпоинт Microsoft Neural TTS. Голос — `en-US-JennyNeural`. API-ключ не нужен.
2. `miniaudio.decode_file(tmp_path)` — декодирует MP3 в сырые PCM-сэмплы.
3. `sounddevice.play(samples, sample_rate, blocking=True)` — воспроизводит синхронно.

---

## 9. База данных — db.py

SQLite хранится в файле `history.db` рядом с проектом. Таблица `messages` создаётся автоматически при первом запуске.

Структура таблицы: `id`, `timestamp` (локальное время по умолчанию), `role` (user или bot), `text`.

Четыре функции:
- `save_message(role, text)` — записывает сообщение
- `get_history(limit)` — последние N сообщений в хронологическом порядке
- `get_today_history()` — только сегодняшние записи
- `clear_history()` — удаляет все записи

---

## 10. Текстовый ввод — gui/main_window.py

При нажатии Enter или кнопки Send запускается `_TextWorker` — отдельный `QThread`. Это нужно, чтобы сетевые запросы (например, погода) не замораживали интерфейс. Воркер вызывает `process_query`, сохраняет оба сообщения в БД и возвращает `display` через сигнал `finished` в основной поток, который добавляет его в чат.

## Итог

Проект демонстрирует применение: объектно-ориентированного программирования (скилл-система), многопоточности (QThread, threading.Timer), работы с внешними API (Groq, OpenWeatherMap, edge-tts), базы данных SQLite и графического интерфейса на PySide6.
